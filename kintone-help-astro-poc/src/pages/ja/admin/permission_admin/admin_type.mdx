<Heading id="permission_admin_permissions_concept_30">アプリ管理者</Heading>

<Heading level={3} id="permission_admin_permissions_concept_3010">アプリ管理者ができること</Heading>

アプリ管理者はアプリごとに存在し、そのアプリの設定を変更できます。また、自分がアプリ管理者になっているアプリを一覧で確認できます。

<Heading level={4} id="permission_admin_permissions_concept_301010">できることの例</Heading>

* フォームの設定
* プロセス管理の設定
* 通知の設定
* アクセス権の設定

<Heading level={3} id="permission_admin_permissions_concept_3020">アプリ管理者が設定できるアクセス権</Heading>

<Heading level={4} id="permission_admin_permissions_concept_302010">アプリに対するアクセス権</Heading>

次の権限を設定できます。

* アプリの管理権限
* レコードの閲覧、追加、編集、または削除
* ファイルの書き出しまたは読み込み

<Heading level={4} id="permission_admin_permissions_concept_302020">レコードに対するアクセス権</Heading>

レコードの条件を設定し、その条件に合致するレコードに対して、閲覧、編集、または削除の権限を設定できます。

<Heading level={4} id="permission_admin_permissions_concept_302030">フィールドに対するアクセス権</Heading>

配置したフィールドに対して、閲覧または編集の権限を設定できます。

<Heading level={3} id="permission_admin_permissions_concept_3030">&lt;Wovn&gt;［アプリの設定］&lt;/Wovn&gt;画面の開きかた</Heading>

<Wovn>［レコードの一覧］</Wovn>画面で、画面右上にある歯車の形をした<Wovn>［アプリを設定］</Wovn>アイコンをクリックします。  
<Img src="/k/kintone/img-ja/app_setting_change_img01.png" alt="スクリーンショット：[アプリを設定]アイコンが枠線で強調されている" />

<Heading level={3} id="permission_admin_permissions_concept_3040">アプリ管理者の確認方法</Heading>

<Wovn>［アプリの設定］</Wovn>の<Wovn>［アプリのアクセス権］</Wovn>画面で、<Wovn>［アプリ管理］</Wovn>が有効になっているユーザーが、アプリ管理者です。初期設定では、アプリ作成者がアプリ管理者になっています。  
必要に応じて管理者を変更できます。詳細については、次のページを参照してください。  
[<Kintone />全体に関わるアクセス権を設定する](/k/ja/id/040867.html)

<Heading id="permission_admin_permissions_concept_40">スペース管理者</Heading>

<Heading level={3} id="permission_admin_permissions_concept_4010">スペース管理者ができること</Heading>

スペース管理者はスペースごとに存在し、そのスペースの設定を変更できます。

<Heading level={4} id="permission_admin_permissions_concept_401010">できることの例</Heading>

* 参加メンバーの管理
* スペースの公開を参加メンバーに限定するかどうかの設定
* 複数のスレッドを使用するかどうかの設定

<Heading level={3} id="permission_admin_permissions_concept_4020">スペース管理者が設定できるアクセス権</Heading>

スペースの公開を参加メンバーに限定するかどうか設定できます。

<Heading level={3} id="permission_admin_permissions_concept_4030">スペースの設定画面の開きかた</Heading>

スペースを開いて、画面右上の<Wovn>［オプション］</Wovn>アイコンをクリックし、<Wovn>［スペースを設定］</Wovn>をクリックします。  
<Img src="/k/kintone/img-ja/permissions_concept_02.png" alt="スクリーンショット：［オプション］アイコンをクリックし、スペースを設定をクリック" />

<Heading level={3} id="permission_admin_permissions_concept_4040">スペース管理者の確認方法</Heading>

スペースの設定画面の<Wovn>［参加メンバー］</Wovn>タブで、<Wovn>［管理者］</Wovn>が有効になっているユーザーが、スペース管理者です。初期設定では、スペース作成者がスペース管理者になっています。  
必要に応じて管理者を変更できます。詳細については、次のページを参照してください。  
[スペースのメンバーを追加／削除する](/k/ja/id/040703.html)

<Heading id="permission_admin_permissions_concept_60">権限の設定例</Heading>

<Heading level={3} id="permission_admin_permissions_concept_6010">アプリを作成および管理できる人を制限したい</Heading>

* アプリの作成や管理を許可したいメンバーにのみ、アプリの作成権限や管理権限を付与します。
* アプリの作成権限や管理権限は、<Kintone />システム管理で設定できます。なお、初期設定では、すべてのユーザーにアプリの作成と管理が許可されています。設定方法については、次のページを参照してください。  
  [<Kintone />全体に関わるアクセス権を設定する](/k/ja/id/040867.html)

<Heading level={3} id="permission_admin_permissions_concept_6020">アプリを削除できる人を制限したい</Heading>

* アプリの削除を許可したいメンバーを、そのアプリの管理者に設定します。
* アプリ管理者は、現在のアプリ管理者が設定できます。なお、初期設定では、アプリ作成者がアプリ管理者になっています。設定方法については、次のページを参照してください。  
  [アプリにアクセス権を設定する](/k/ja/id/040589.html)

なお、アプリ管理者に設定すると、アプリの設定変更も可能になります。十分検討した上で設定してください。

<Heading level={3} id="permission_admin_permissions_concept_6030">アプリの設定変更の権限を別のメンバーに移譲したい</Heading>

* アプリの設定変更を任せたいメンバーを、そのアプリの管理者に設定します。
* アプリ管理者は、現在のアプリ管理者が設定できます。なお、初期設定では、アプリ作成者がアプリ管理者になっています。設定方法については、次のページを参照してください。  
  [アプリにアクセス権を設定する](/k/ja/id/040589.html)

<Heading level={3} id="permission_admin_permissions_concept_6040">アプリのデータをファイルに書き出せる人を制限したい</Heading>

* ファイルの書き出しを許可したいメンバーにのみ、そのアプリのファイル書き出し権限を付与します。
* ファイル書き出し権限は、アプリ管理者が設定できます。なお、初期設定では、アプリ作成者にのみファイル書き出しが許可されています。設定方法については、次のページを参照してください。  
  [アプリにアクセス権を設定する](/k/ja/id/040589.html)

<Heading level={3} id="permission_admin_permissions_concept_6050">&lt;Kintone /&gt;にプラグインを登録できる人を決めておきたい</Heading>

* プラグインの登録を許可したいユーザーを、<Kintone />システム管理者に設定します。
* <Kintone />システム管理者は、現在の<Kintone />システム管理者が設定できます。設定方法については、次のページを参照してください。  
  [<Kintone />全体に関わるアクセス権を設定する](/k/ja/id/040867.html)

なお、<Kintone />システム管理者に設定すると、<Kintone />のシステム全体の設定変更も可能になります。十分検討した上で設定してください。

<Enabled regions={["JP", "US"]}>

<Heading level={3} id="permission_admin_permissions_concept_6060">サポートに問い合わせできる人を追加したい</Heading>

* 問い合わせを許可したいメンバーを、<Slash />共通管理者に設定します。
* <Slash />共通管理者は、現在の<Slash />共通管理者が設定できます。設定方法については、次のページを参照してください。  
  [<Slash />共通管理者の設定](/general/ja/id/0208.html)

なお、<Slash />共通管理者に設定すると、ユーザーの追加や削除、セキュリティなどの設定変更も可能になります。十分検討した上で追加してください。

</Enabled>
