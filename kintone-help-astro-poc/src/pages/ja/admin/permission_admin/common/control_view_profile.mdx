<SlashUi />で、所属している最上位組織が異なるユーザーのプロフィール画面と組織情報へのアクセスを制限できます。この機能を、組織間のアクセス権の設定と呼びます。  

<Img src="/k/kintone/img-ja/control_view_profile_img1.png" alt="組織間のアクセス権の設定" />

[<SlashUi />を開く](/k/ja/id/04047.html)  

<Heading id="common_control_view_profile_10">プロフィールの閲覧制限</Heading>

組織間のアクセス権の設定を有効にすると、所属する最上位組織が異なるユーザー間では、互いのプロフィールと組織情報を閲覧できなくなります。  
所属している最上位組織が同じであれば、所属する組織の階層に関わらず、互いのプロフィールと組織情報を閲覧できます。  
<Img src="/k/kintone/img-ja/control_view_profile_img2.png" alt="プロフィールの閲覧制限" />  
ただし、<Slash />共通管理者は、<SlashUi />で、各ユーザーのプロフィールと組織情報を閲覧できます。

<Heading id="common_control_view_profile_20">検索の制限</Heading>

組織間のアクセス権の設定を有効にすると、所属する最上位組織が異なるユーザーと組織を検索できなくなります。  
また、コメントの宛先指定でも、所属する最上位組織が異なるユーザーは指定できません。  
そのほかにも、所属する最上位組織が異なるユーザーが画面に表示されなくなるなど、ユーザーや組織の情報へのアクセスが制限されます。  
詳細は、次の項目を参照してください。  
[組織間のアクセス権の設定で制限される表示と選択](#common_control_view_profile_30)

<Note>

組織間のアクセス権の設定方法は、<SlashHelp />を参照してください。  
[組織間のアクセス権の設定](/general/ja/admin/list_useradmin/list_division/dep_accessctrl.html)

</Note>

<Heading id="common_control_view_profile_30">組織間のアクセス権の設定で制限される表示と選択</Heading>

組織間のアクセス権の設定を有効にした場合に制限される操作を、画面ごとに説明します。  
たとえば、<Slash />共通管理の表では、組織間のアクセス権の設定を有効にした場合、一般ユーザーは組織の異なるユーザーのプロフィールを閲覧できないことを示しています。  

制限される表示と選択は、PCとモバイルで同様です。
