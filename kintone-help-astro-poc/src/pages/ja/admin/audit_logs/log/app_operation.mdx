アプリのレコードの操作をしたときに出力されるログについて説明します。

<Note>

ユーザーが操作するアプリの環境に応じて、「アクション」の末尾に次の値が出力されます。

* 運用環境の場合：(Public environment)
* 動作テスト環境の場合：(Test environment)

</Note>

<Heading level={4} id="log_app_operation_106520">レベル</Heading>

情報

<Heading level={4} id="log_app_operation_106530">「補足」に出力される内容</Heading>

次の項目が出力されます。

* アプリID
* アプリ名
* レコード番号
* 通知ID
* イベントタイプ
* 送信先サーバーのURL

通知に成功した場合は、次の項目が出力されます。

* 通知のステータスコード

<Kintone />のエラーで通知に失敗した場合は、次の項目が出力されます。

* エラータイプ：CLIENT_ERROR
* エラーメッセージ

通知を受け取るWebサービスのエラーで通知に失敗した場合は、次の項目が出力されます。

* エラータイプ：SERVER_ERROR
* 通知のHTTPステータスコード

<Note>

レコードのコメントを書き込んだ場合、コメント番号（comment id: *）がログに書き出されます。

</Note>

<Heading level={4} id="log_app_operation_106540">出力されるログの例</Heading>

<Heading level={5} id="log_app_operation_10654010">通知に成功した場合</Heading>

```
app id: *, app name: *, record id: *, notification id: *, event type: {ADD_RECORD / ADD_RECORD_COMMENT / UPDATE_RECORD / UPDATE_STATUS / DELETE_RECORD}, server url: *, status code: *

```

<Heading level={5} id="log_app_operation_10654020">&lt;Kintone /&gt;のエラーで通知に失敗した場合</Heading>

```
app id: *, app name: *, record id: *, notification id: *, event type: {ADD_RECORD / ADD_RECORD_COMMENT / UPDATE_RECORD / UPDATE_STATUS / DELETE_RECORD}, server url: *, error type: CLIENT_ERROR, error message: *

```

<Heading level={5} id="log_app_operation_10654030">通知を受け取るWebサービスのエラーで通知に失敗した場合</Heading>

```
app id: *, app name: *, record id: *, notification id: *, event type: {ADD_RECORD / ADD_RECORD_COMMENT / UPDATE_RECORD / UPDATE_STATUS / DELETE_RECORD}, server url: *, error type: SERVER_ERROR, status code: *

```

<Heading id="log_app_operation_20">プロパティ</Heading>

監査ログの「補足」欄に出力されるプロパティは次のとおりです。

<Heading level={3} id="log_app_operation_2010,class=&quot;no-trans&quot;">app id</Heading>

アプリのIDです。

<Heading level={3} id="log_app_operation_2012,class=&quot;no-trans&quot;">app name</Heading>

アプリ名です。

<Heading level={3} id="log_app_operation_2014,class=&quot;no-trans&quot;">comment id</Heading>

レコードのコメント番号です。

<Heading level={3} id="log_app_operation_2015,class=&quot;no-trans&quot;">Email</Heading>

メールアドレスです。

<Heading level={3} id="log_app_operation_2016,class=&quot;no-trans&quot;">error message</Heading>

エラーメッセージです。

<Heading level={3} id="log_app_operation_2018,class=&quot;no-trans&quot;">error type</Heading>

エラータイプです。次のエラータイプが出力されます。

* CLIENT_ERROR：<Kintone />が原因で通知の送信に失敗している場合に出力されます。
* SERVER_ERROR：通知を受け取るWebサービス側のエラーが原因で失敗している場合に出力されます。

<Heading level={3} id="log_app_operation_2020,class=&quot;no-trans&quot;">event type</Heading>

イベントタイプです。次のイベントタイプが出力されます。

* ADD_RECORD：レコードの追加
* ADD_RECORD_COMMENT：レコードコメントの追加
* UPDATE_RECORD：レコードの更新
* UPDATE_STATUS：プロセス管理のステータスの更新
* DELETE_RECORD：レコードの削除

<Heading level={3} id="log_app_operation_2022,class=&quot;no-trans&quot;">file size</Heading>

ファイルサイズです。

<Heading level={3} id="log_app_operation_2024,class=&quot;no-trans&quot;">filename</Heading>

ファイル名です。

<Heading level={3} id="log_app_operation_2026,class=&quot;no-trans&quot;">login name</Heading>

ゲストユーザーのログイン名です。

<Heading level={3} id="log_app_operation_2028,class=&quot;no-trans&quot;">notification id</Heading>

通知IDです。

<Heading level={3} id="log_app_operation_2030,class=&quot;no-trans&quot;">number of file lines</Heading>

ファイルの行数です。

<Heading level={3} id="log_app_operation_2032,class=&quot;no-trans&quot;">record id</Heading>

レコード番号です。

<Heading level={3} id="log_app_operation_2034,class=&quot;no-trans&quot;">server url</Heading>

サーバーURLです。

<Heading level={3} id="log_app_operation_2036,class=&quot;no-trans&quot;">slack subdomain</Heading>

通知を受信するSlackのサブドメインです。

<Heading level={3} id="log_app_operation_2038,class=&quot;no-trans&quot;">status code</Heading>

ステータスコードです。

<Heading level={3} id="log_app_operation_2040,class=&quot;no-trans&quot;">user</Heading>

ログイン名です。
