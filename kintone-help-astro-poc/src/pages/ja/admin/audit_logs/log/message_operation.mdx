メッセージの操作をしたときに出力されるログについて説明します。

<Heading id="log_message_operation_10,class=&quot;no-trans&quot;">Message operation</Heading>

モジュールが「Message operation」である監査ログは、下記のとおりです。

<Heading level={3} id="log_message_operation_1010,class=&quot;no-trans&quot;">Message comment file download</Heading>

アクションが「Message comment file download」である監査ログについて説明します。

<Heading level={4} id="log_message_operation_101010">実行された操作</Heading>

メッセージに添付されたファイルについて、次の操作をした。

* ファイルをダウンロードした
* 画像のスライドショーを表示した

<Heading level={4} id="log_message_operation_101020">レベル</Heading>

情報

<Heading level={4} id="log_message_operation_101030">「補足」に出力される内容</Heading>

次の項目が出力されます。

* ログイン名
* メッセージのURL
* ファイル名

<Heading level={4} id="log_message_operation_101040">出力されるログの例</Heading>

```
users: [*], comment url: *, filename: *

```

ログイン名はカンマ区切りで出力されます。

<Heading id="log_message_operation_20">プロパティ</Heading>

監査ログの「補足」欄に出力されるプロパティは次のとおりです。

<Heading level={3} id="log_message_operation_2010,class=&quot;no-trans&quot;">comment url</Heading>

メッセージのURLです。

<Heading level={3} id="log_message_operation_2020,class=&quot;no-trans&quot;">filename</Heading>

ファイル名です。

<Heading level={3} id="log_message_operation_2030,class=&quot;no-trans&quot;">users</Heading>

ユーザーのログイン名です。
