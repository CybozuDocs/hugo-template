スペースの操作をしたときに出力されるログについて説明します。

<Note>

ゲストユーザーが操作をした場合は、ログの先頭にゲストユーザーのログイン名（login name: *）が出力されます。

</Note>

<Heading id="log_space_10,class=&quot;no-trans&quot;">Space management</Heading>

モジュールが「Space management」である監査ログは、下記のとおりです。

<Heading level={3} id="log_space_1010,class=&quot;no-trans&quot;">Space add</Heading>

アクションが「Space add」である監査ログについて説明します。

<Heading level={4} id="log_space_101010">実行された操作</Heading>

スペース、またはゲストスペースを作成した。

<Heading level={4} id="log_space_101020">レベル</Heading>

情報

<Heading level={4} id="log_space_101030">「補足」に出力される内容</Heading>

次の項目が出力されます。

* スペースID
* スペース名

<Heading level={4} id="log_space_101040">出力されるログの例</Heading>

```
space id: *, space name: *

```

<Heading level={3} id="log_space_1015,class=&quot;no-trans&quot;">Space update</Heading>

<Heading level={4} id="log_space_102010">実行された操作</Heading>

スペースを削除した。

<Note>

<Kintone />システム管理画面の[スペース管理]でスペースを削除した場合も出力されます。

</Note>

<Heading level={4} id="log_space_102020">レベル</Heading>

情報

<Heading level={4} id="log_space_102030">「補足」に出力される内容</Heading>

次の項目が出力されます。

* スペース名
* アプリID
* アプリ名

<Heading level={4} id="log_space_102040">出力されるログの例</Heading>

<Heading level={5} id="log_space_10204010">スペース内にアプリが配置されていなかった場合</Heading>

```

```

<Heading level={5} id="log_space_10204020">スペース内にアプリが配置されていた場合</Heading>

```
space id: *, space name: *, (app id: *, app name: *), (...

```

先頭には削除対象のスペース、（）内には同時に削除されたアプリが出力されます。

<Heading level={3} id="log_space_4024,class=&quot;no-trans&quot;">space template id</Heading>

<Heading level={3} id="log_space_4026,class=&quot;no-trans&quot;">space template name</Heading>

スペーステンプレート名です。

<Heading level={3} id="log_space_4028,class=&quot;no-trans&quot;">thread id</Heading>

スレッドのIDです。

<Heading level={3} id="log_space_4030,class=&quot;no-trans&quot;">thread name</Heading>
