アプリの設定に関する操作をしたときに出力されるログについて説明します。

<Heading id="log_app_management_10,class=&quot;no-trans&quot;">App management</Heading>

モジュールが「App management」である監査ログは、下記のとおりです。

<Heading level={3} id="log_app_management_1010,class=&quot;no-trans&quot;">App update</Heading>

アクションが「App update」である監査ログについて説明します。

<Heading level={4} id="log_app_management_101010">実行された操作</Heading>

* 「アプリの設定」で設定を変更した
* 「アプリの設定」の「高度な設定」で、レコードのコメント機能の設定を変更した
* 「アプリの設定」の「高度な設定」で、レコードの変更履歴機能の設定を変更した
* 「アプリの設定」の「高度な設定」で、レコードの再利用機能の設定を変更した
* 「アプリの設定」の「高度な設定」で、レコードの一括削除機能の設定を変更した
* 「アプリの設定」の「高度な設定」で、アプリコードの設定を変更した
* 「アプリの設定」の「高度な設定」で、レコード一覧でのインライン編集機能の設定を変更した

<Heading level={4} id="log_app_management_101020">レベル</Heading>

次の操作は「重要」と出力されます。

* 「アプリの設定」の「高度な設定」で、レコードのコメント機能の設定を変更した
* 「アプリの設定」の「高度な設定」で、レコードの変更履歴機能の設定を変更した

次の操作は「情報」と出力されます。

* 「アプリの設定」で設定を変更した
* 「アプリの設定」の「高度な設定」で、レコードの再利用機能の設定を変更した
* 「アプリの設定」の「高度な設定」で、レコードの一括削除機能の設定を変更した
* 「アプリの設定」の「高度な設定」で、アプリコードの設定を変更した
* 「アプリの設定」の「高度な設定」で、レコード一覧でのインライン編集機能の設定を変更した

<Heading level={4} id="log_app_management_101030">「補足」に出力される内容</Heading>

<Heading level={5} id="log_app_management_10103010">「アプリの設定」で設定を変更した場合</Heading>

次の項目が出力されます。

* アプリID
* アプリ名
* 変更した設定項目（target）  
  詳細は、次の項目を参照してください。  
  [target](#log_app_management_7040)

<Heading level={5} id="log_app_management_10103020">「高度な設定」で該当する設定を変更した場合</Heading>

次の項目が出力されます。

* アプリID
* アプリ名
* 変更した設定項目と状態

変更した設定項目は、下記が出力されます。

* レコードのコメント機能（record comment）
* レコードの変更履歴機能（record history）
* レコードの再利用機能（record duplication）
* レコードの一括削除機能（bulk delete）
* アプリコードの設定（app code）
* レコード一覧でのインライン編集機能（record inline edit and delete）

変更した設定項目の状態は、下記のどちらかです。

* 有効にした：true
* 無効にした：false

<Note>

変更した設定項目がアプリコードの設定（app code）の場合、状態は出力されません。

</Note>

<Heading level={4} id="log_app_management_101040">出力されるログの例</Heading>

```
app id: *, app name: *, target: *

```

```

<Heading level={5} id="log_app_management_10104040">レコードの再利用機能の設定を変更した場合</Heading>

```

<Heading level={3} id="log_app_management_7040,class=&quot;no-trans&quot;">target</Heading>

「アプリの設定」で、設定を変更した箇所が出力されます。

* form：フォームの設定
* view：一覧を追加、変更、または複製
* report：グラフを追加、変更、または複製
* general：アプリ名、またはアイコンと説明
* icon：アプリアイコン  
  「アプリの設定」画面で変更した場合に出力されます。  
  「一般設定」の「アイコンと説明」画面でアプリアイコンを変更した場合は「general」が出力されます。
* theme：デザインテーマ
* status：プロセス管理
* notification：アプリの条件通知、レコードの条件通知、リマインダーの条件通知
* plugin：プラグイン
* customize：JavaScript/CSSでカスタマイズ
* api token：APIトークン
* webhook：Webhook
* app acl：アプリのアクセス権
* record acl：レコードのアクセス権
* field acl：フィールドのアクセス権
* category：カテゴリー
* resource：言語ごとの名称
* title：レコードのタイトル
* info：高度な設定
* action：アクション
* maintenance：メンテナンスモード  
  開始時に「maintenance: enabled」が出力されます。  
  終了時に「maintenance: disabled」が出力されます。
