<Kintone />のディスク使用量を減らすには、不要な添付ファイルを削除してください。  
添付ファイルを削除すると、その分だけディスク使用量が減り、新たなデータを追加したときに空き容量を使用できます。  

ここでは、ディスク使用量を減らす方法とディスク使用量の考え方について説明します。  

<Heading id="disk_data_disk_size_10">添付ファイルのディスク使用量を減らす</Heading>

<Heading level={3} id="disk_data_disk_size_1000">アプリやスレッドなどの添付ファイルを削除する</Heading>

添付ファイルを削除すると、ディスク使用量の空き容量が増えます。  
ただし、変更履歴が有効となっているアプリでは、変更履歴内にも添付ファイルが保存されています。  
そのため、レコードから添付ファイルを削除しただけではディスク使用量は減りません。変更履歴内の添付ファイルを削除する必要があります。  
[アプリの変更履歴内の添付ファイルを削除する](#disk_data_disk_size_1010)

<Hint>

ディスク使用量を減らす最も効果的な方法は、アプリの添付ファイルの削除です。  
添付ファイルのサイズが大きいアプリを見つけるには、アプリ一覧の<Wovn>［添付ファイル］</Wovn>列を利用すると便利です。  
[アプリごとの添付ファイルサイズの合計を確認する](/k/ja/id/040773.html)  

</Hint>

<Heading level={3} id="disk_data_disk_size_1010">アプリの変更履歴内の添付ファイルを削除する</Heading>

アプリの変更履歴内の添付ファイルも、ディスク使用量に含まれます。  
変更履歴内の添付ファイルを削除するには、次のいずれかの操作が必要です。

* レコードを削除する
* <Wovn>［添付ファイル］</Wovn>フィールドを削除する
* レコードの変更履歴の記録機能を無効にする

添付ファイルの変更履歴がない場合は、レコードの変更履歴の記録機能を無効にしても、ディスク使用量は減りません。  
レコードから添付ファイルを削除する必要があります。  

レコードの変更履歴を無効に設定すると、これまでの履歴は削除されます。  
削除には時間がかかる場合があるため、すぐにはディスク使用量が減らないことがあります。  
[レコードの変更履歴の記録機能を無効にする](/k/ja/id/040460.html)

<Warning>
API を使用する場合など、レコードを頻繁に更新する場合は、変更履歴の記録機能を無効にすることを推奨します。  
有効にしていると、ディスク使用量が急増する場合があります。

</Warning>

<Heading id="disk_data_disk_size_20">監査ログのディスク使用量を減らす</Heading>

APIを使用する場合など、大量のデータの追加／更新／削除を頻繁に繰り返すと、監査ログが多く記録されます。添付ファイルを削除したり、レコードの変更履歴を無効にしたりしてもディスク使用量が減らない場合は、監査ログの保存期間を短くすると、ディスク使用量が減る場合があります。  
[監査ログの保存期間を設定する](/general/ja/id/020104.html)

<Heading id="disk_data_disk_size_30">参考：ディスク使用量の考え方</Heading>

<SlashUi />の<Wovn>［契約状況］</Wovn>に表示されるディスク使用量は、添付ファイル領域、監査ログ保存領域、およびデータベース領域を合計した値です。  
<SlashUi />の<Wovn>［契約状況］</Wovn>には、利用中のドメイン全体でのディスク使用量と、サービスごとのディスク使用量が表示されます。  
詳細は、次のページを参照してください。  
[契約状況](/general/ja/id/02078.html)

<Heading level={3} id="disk_data_disk_size_3010">添付ファイル領域のデータ</Heading>

添付ファイル領域に含まれるデータは、アプリの<Wovn>［添付ファイル］</Wovn>フィールドや、スレッドのコメントなどに添付されたファイルです。添付ファイル領域に含まれるデータを削除すると、ディスク使用量が減り、空き容量が増えます。

<Img src="/k/kintone/img-ja/disk_size_img01.png" alt="添付ファイル領域のデータ" />

<Heading level={3} id="disk_data_disk_size_3020">データベース領域のデータ</Heading>

データベース領域に含まれるデータは、次のとおりです。

* 添付ファイル以外の入力データ
* 設定情報

データベース領域は、保存データに応じて自動的に拡張されます。一度拡張された領域は保存データの有無にかかわらず減らないため、データベース領域のデータを削除してもディスク使用量は変わりません。データを削除したデータベースの空き容量は、再利用されます。
<Img src="/k/kintone/img-ja/disk_size_img02.png" alt="データベース領域のデータ" />

<Heading level={3} id="disk_data_disk_size_3030">監査ログ領域のデータ</Heading>

監査ログのデータ保存領域です。監査ログの保存期間を短くすることで、ディスク使用量が減り、空き容量が増える場合があります。
